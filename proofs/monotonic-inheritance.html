OS-006 Delegatic kernel

⊆ — Monotonic Inheritance

Proved

A team cannot grant its members a capability the team itself does not hold. Every child context inherits a strict subset of its parent's permissions, and any attempt to escalate beyond the parent is rejected before it is written. Privilege only ever narrows as you descend the delegation tree.

01The claim

Let perms(c) be the permission set of context c with parent p. The invariant is a subset law on the delegation lattice:

∀ child c with parent p: perms(c) ⊆ perms(p) Attempted escalation (perms(c) ⊄ perms(p)) is rejected at write time. Auditing the root bounds everything reachable downstream.

02The evidence

delegatic/ v0.1.0 · authorization kernel

The kernel computes the effective permission set as a lattice meet against the parent and refuses any grant that would exceed it. Covered by the 42-test kernel suite, including the privilege-escalation rejection cases.

workbench/app/src/lib/ia/ · L14 (consume soundness)

gate.authority projects an over-broad child grant into a value whose σ (conflict set) is non-empty, so consume(value, { sigma_empty }) refuses it. Law L14 guarantees the gate only passes when σ is genuinely empty — i.e. when inheritance really was monotonic.

03Verify it yourself

cd delegatic && mix test # escalation rejected at write time cd workbench/app npm test -- ia/laws.test.ts # L14 — consume soundness backs gate.authority

04What authority ⊆ grants

⊆ grants privilege containment. Because authority only ever shrinks downward, the blast radius of any delegated context is bounded by its ancestors — you never have to read the whole tree to know what a leaf can do. It is the lattice complement to ⊘ (no cycles): one bounds depth, the other bounds breadth.
