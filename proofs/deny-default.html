⊥ — Deny by Default

Proved

An action is permitted only if an explicit grant exists for it. Silence is refusal. There is no ambient authority, no implicit "allow unless denied" — the policy surface is fully explicit, so you can read what is allowed instead of guessing what isn't forbidden.

01The claim

permit(action) ⟺ ∃ explicit grant g such that g authorizes action No grant ⟹ ⊥ (refuse). The default of the policy lattice is bottom, not top.

This is standard capability-system discipline — authority is a possessed token, never an inferred default — applied to agent governance. It is what makes the policy surface auditable: every "yes" has a citable grant behind it.

02The evidence

delegatic/ v0.1.0 · authorization kernel

The Authorizer returns a refusal for any action lacking a matching grant in the PolicyStore; only a matching, HMAC-signed AuthorizationBlock yields a permit. The 42-test kernel suite includes the no-grant-refuses cases.

workbench/app/src/lib/ia/ · L14 (consume soundness)

In the Workbench, an undeclared capability surfaces as a non-empty σ on the authority value, so gate.authority fails it — exactly the negative case verified end-to-end (an injected undeclared &body.os capability fails no_hidden_capability + authority). Law L14 guarantees the pass verdict is never granted to an ungranted action.

03Verify it yourself

cd delegatic && mix test # absence of grant ⟹ refusal cd workbench/app npm test -- ia/laws.test.ts # L14 — consume soundness backs gate.authority

04What authority ⊥ grants

⊥ grants the guarantee that an explicit grant is required for every action — eliminating the most common source of agent privilege bugs, the implicit allow. Together with ⊆ (monotonic) it means the only capabilities in play are the ones someone deliberately wrote down.
