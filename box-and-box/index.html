box-and-box

The complete governance kernel — eight modal rungs, one bridge (feasible ▸ permitted ▸ best), and 97 property-tested laws. Pure arithmetic: no LLM, no network, no I/O. Read every line below, then take it to your own language.

# the published package — read it, run it, port it
npm install -g box-and-box
box-and-box laws   # the 97-law conformance harness (2000 trials/law)
box-and-box govern decision.json # a real verdict → certificate JSON

The arithmetic is deliberately language-agnostic. Here is the heart of rung 1 — consume(value, requirements), the boolean gate that every higher rung sits on top of — written five ways.

JavaScript is the conformance reference. A port is certified by passing test/laws.mjs (all 97 laws, 2000 trials each) — not by looking right. The four non-JS snippets are faithful, idiomatic translations of the canonical source to show the shape travels; they are illustrative until their language runs the law suite. (The bundled Elixir host, aios/box_and_box_aios.ex, follows the same rule — it delegates to the verified JS CLI for real verdicts.)
