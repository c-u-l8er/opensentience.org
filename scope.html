SCOPE — awareness for typed regions, across substrates.

One spatial protocol pair. A Region algebra says where in any substrate — geo, grid, document, graph. A typed SpatialClaim envelope says who is doing what, how hard, until when. Fifteen multi-agent coordination dimensions collapse into two cuts — and the GeoFleetic ↔ RuneFort bridge falls out for free.

I.

The collapse: 15 dimensions → 2 gaps

the architectural finding

The cleanest result of the spatial-coordination research: fifteen things the multi-agent literature treats as separate problems are the same primitive at different time horizons and coupling strengths. Naming the two underlying gaps is cheaper than shipping fifteen protocols — and the unification is what nobody else has.

Gap A · the verb layer

SpatialClaim envelope

A typed assertion of intent over a region. Carries kind (presence, edit, reserve, territory, coverage, memory, handoff), strength (advisory · soft · exclusive), a time horizon, confidence, and a supersedes chain for conflict resolution.

Carried over CloudEvents v1 — the same envelope PULSE already uses for cross-loop tokens.

Subsumes 6: shared spatial state · intent broadcast · conflict resolution · presence · attention/salience · territory/role
Gap B · the noun layer

N-D Region algebra

A discriminated union of region kinds with substrate-agnostic operations: contains · overlaps · distance · union · buffer · project.

Allen interval algebra (temporal), RCC-8 (geographic), and Manhattan (grid) all exist separately today. The unification across substrates is what's missing.

Subsumes 4: cognitive viewports · spatial memory tagging · N-D substrates · spatio-temporal traces

The unifying SpatialClaim envelope

type SpatialClaim = {
  agent:      AgentId,
  region:     Region,            // the noun — see Gap B
  kind:       ClaimKind,         // presence | edit | reserve | territory | coverage | memory | handoff
  strength:   Strength,          // advisory | soft | exclusive
  horizon:    TimeHorizon,       // "active during phase X" / "valid until window Y"
  confidence: [0, 1],
  supersedes: ClaimId[],         // conflict-resolution chain
  trace_id:   TraceId             // CloudEvents v1 correlation
}

type Region = { kind: "grid" | "document_span" | "graph_subgraph", ... }
              // v0.1 minimum — geo / file_range / viewport deferred to v0.2
II.

One region type, every substrate

the bridge nobody built

Today every portfolio piece picks its own region type and they never interoperate. SCOPE makes them kinds of one Region. The long-hand-wavy GeoFleetic ↔ RuneFort synergy becomes a concrete, two-line bridge.

Region kind Maps onto v0.1 status
grid RuneFort layout primitives — rooms / claims / neighbors over an integer grid in v0.1
document_span BendScript graph regions — typed inline link facets over bend:doc#blk.spn URIs in v0.1
graph_subgraph Graphonomous — region-scoped retrieval over graph hops in v0.1
geo GeoFleetic GeoJSON polygons / geofences v0.2
document Shared whiteboard / canvas — multiple agents emitting presence + intent + territory claims v0.2

The bridge, concretely: a fleet asset entering a geofence (GeoFleetic) emits a SpatialClaim{kind: presence, region: geo}, which a RuneFort renderer displays as a state binding on the corresponding grid tile (region: grid). A geographic region and a grid-cell room become two kinds of the same type. That's the test that A + B are the right cuts: the "shared whiteboard for LLM agents" falls out with no new primitive — a document region plus presence / intent / attention / territory claims, with Deliberatic for hard conflicts and Graphonomous persisting claims as semantic nodes.

III.

What v0.1 commits to — consumer-gated

honest surface area

Architecture review against the stack and the competitive landscape (tf2, W3C Web Annotation, MAPF/CBS, Yjs awareness, etcd leases) found the draft's surface area ~40% larger than current consumers justify. v0.1 ships only what a real consumer needs today; the rest defers with explicit commitments.

Region kinds
3 in v0.1 grid, document_span, graph_subgraph. deferred file_range (no range-level consumer), viewport (no multi-agent-per-viewport consumer yet).
Claim kinds
6–7 in v0.1 presence, edit, reserve, territory, coverage, memory, handoff. deferred avoid/hazard → scope.robotics.v1. Dropped attention (overlaps presence), observation/drift (already PULSE OutcomeSignal / SurpriseSignal).
Strength levels
3 in v0.1 advisory, soft, exclusive. deferred mandatory — until a fencing-token mechanism exists (the etcd / Chubby / Redlock lesson: without monotonic claim versions per (frame, region), stale clients act after expiry).
Subscription
promoted to v0.1 The whole stack is event-driven (PULSE / Graphonomous / PRISM are all CloudEvents-based). Without subscription, agents must poll a static document — so it's the core feature of a coordination protocol, not a v0.2 nicety.
IV.

Hard prerequisites before v0.1 final

what must ship first

SCOPE doesn't stand alone. Two dependencies are blocking and documented as such — naming them honestly is part of the protocol-extraction discipline.

blocking
PULSE v0.1.2 vendor-namespace tokens. PULSE v0.1.1's closed token enum cannot accept scope.v1.* tokens. The vendor-namespace mechanism must be specified before SCOPE can be normatively conformant.
blocking
Graphonomous region field. The Node schema has no region field today. Region-scoped retrieval is the correct division of labor, but it's a parallel Graphonomous-side dependency that SCOPE v0.1 must document.
field naming
Reconcile horizon (richer time-window: "active during phase X" / "valid until window Y") vs. the draft's expires_at (single timestamp) before v0.1 final. horizon wins — it maps cleaner onto PULSE phases.
[&] framing
SCOPE introduces region and claim as new &space.* subtypes (today: fleet / route / geofence) — backward-compatible, not a "v0.2 maturation."
V.

Prior art — two genuine novelties, the rest assembly

honest positioning

SCOPE's two real inventions are substrate-agnostic frames (lifted from tf2, generalized beyond rigid-body transforms) and substrate-agnostic typed claims with a strength gradient (synthesized from etcd/Chubby + Yjs awareness + MAPF). Everything else is careful assembly — and SCOPE cites it.

ROS REP 105 / tf2 — frames W3C Web Annotation — selectors via refinedBy MAPF / Conflict-Based Search — typed conflicts Yjs awareness · Liveblocks presence — ephemeral TTL claims etcd leases · Chubby — advisory locks + strength

The empty seat is real: ROS2 tf2 is poses-only, DDS-RTPS QoS is topic-bound, Liveblocks/Yjs awareness is untyped, Figma multiplayer is proprietary cursors, the Contract Net Protocol is an algorithm without a carrier, and MASCN (AAMAS 2025) is a concept paper. The recommended pitch — "awareness for typed regions, across substrates" — names the analog every collaborative-software builder already has cached, then the three concrete extensions (typed regions, frames, strengths) that justify a new protocol.

VI.

Two primitives, every coordinated space

by domain

A Region and a typed SpatialClaim are the only two things SCOPE adds. Below they wear five domains — a warehouse grid, a drone corridor, a city geofence, a shared canvas, an AR anchor. Same two primitives; the kind and the strength are all that change.

AMR floor coordination. A robot reserves the cells it's about to cross as a mandatory territory claim with a fencing token and a short lease. Overlaps are hard conflicts — Deliberatic decides; grants persist to Graphonomous for replay.

 warehouse_floor_space.ex Elixir / SCOPE.Space
region grid claim territory · mandatory conflict → Deliberatic

UAS airspace deconfliction. A drone files its corridor as an advisory intent ahead of time, then upgrades to a mandatory presence claim with a heartbeat TTL on entry. The region is a 3-D geo volume, not a flat polygon.

 airspace_corridor.ex Elixir / SCOPE.Space
region geo · 3-D volume claim intent → presence strength advisory → mandatory

The cross-substrate bridge. An asset crossing a geofence emits a presence claim over a geo region — and the same claim re-binds onto a RuneFort grid tile. A geographic region and a grid cell become two kinds of one Region.

 fleet_geofence_presence.ex Elixir / SCOPE.Space
region geo → grid claim presence bridge GeoFleetic ↔ RuneFort

Shared canvas for agents. The "multiplayer whiteboard for LLMs" falls out with no new primitive: a document region plus presence and attention claims. Soft claims converge CRDT-style; only territory escalates to Deliberatic.

 world_awareness.ex Elixir / SCOPE.Space
region document claim attention + presence conflict CRDT converge

Shared AR anchors. Two people looking at the same table: a gaze is attention (advisory, 250 ms), placing an object is territory (mandatory). The region is a pose-anchored geo volume; grants persist to Graphonomous.

The conformance surface is the Region + SpatialClaim schema. v0.1 ships grid · document_span · graph_subgraph; the geo and document kinds shown here are the committed v0.2 extensions.

Status: v0.1 draft, consumer-gated revision in progress

A SCOPE v0.1 draft exists (sections 0–17). The next move is not green-field spec work — it's consumer-gated scope reduction plus a subscription contract, gated on PULSE v0.1.2 and a Graphonomous region field. SCOPE delivers six-plus coordination dimensions at once and is cheaper than the six protocols it replaces; it sequences after the half-built primitives are finished, per the stack's "finish what's half-built first" principle.

Engineering risks the draft must answer, paid-for lessons from production predecessors: fencing tokens for mandatory (Kleppmann via etcd/Chubby), multiple selectors per anchor for document_span drift (W3C refinedBy, not single anchors), UTF-16/UTF-8/grapheme encoding negotiation (LSP PositionEncodingKind), logical clocks for expires_at skew, tombstone GC for superseded chains, a binary wire profile for >1 Hz presence, and signed claims (A2A v1.0 Signed Agent Cards — unsigned claims are spoofable).
